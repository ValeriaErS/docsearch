# Docsearch— система поиска по документации с использованием RAG

# Общие сведения
Программный комплекс предназначен для загрузки корпуса документов, выполнения семантического поиска по фрагментам текста и предоставления пользователю релевантных ответов на естественно-языковые запросы. Система реализована в виде утилиты командной строки с поддержкой индексации документов, векторного поиска и интеграции с локальными языковыми моделями через LM Studio.

# Состав программного комплекса
cmd/docsearch/ — точка входа, обработка команд CLI;

internal/corpus/ — загрузка и нормализация документов;

internal/chunk/ — разбиение текста на фрагменты с перекрытием;

internal/embed/ — вычисление эмбеддингов через LM Studio;

internal/store/ — векторное хранилище;

internal/retrieve/ — поиск топ‑k похожих чанков по косинусной близости;

internal/rag/ — основная логика RAG-пайплайна;

configs/config.yml — конфигурационный файл;

docs/ — каталог с документацией для индексации.

# Назначение системы
Система обеспечивает:

индексацию документов в форматах .md и .txt;

нормализацию текста и разбиение на чанки с перекрытием;

вычисление эмбеддингов через локальную модель в LM Studio;

хранение векторов в оперативной памяти;

поиск топ‑k релевантных фрагментов по косинусной близости;

проверку порога релевантности min_score и сообщение об отсутствии ответа;

формирование связного ответа через LLM со ссылками на источники.

# Руководство системного программиста
Требования к окружению:
Go 1.20 или выше;
LM Studio с загруженной моделью эмбеддингов (nomic-embed-text-v1.5);
свободный порт 1234 для локального сервера LM Studio.

# Запуск системы
Перед запуском убедись, что LM Studio запущена, модель эмбеддингов загружена и локальный сервер включён.

Индексация документов:

bash
go run cmd/docsearch/main.go --index

Поиск по запросу:

bash
go run cmd/docsearch/main.go --query "Что такое эмбеддинги?"

Сборка исполняемого файла:

bash
go build -o bin/docsearch.exe cmd/docsearch/main.go

Запуск собранного файла:

bash
bin/docsearch.exe --query "текст запроса"

# Формат входных данных
Система поддерживает файлы с расширениями .md и .txt. Каждый документ нормализуется, после чего разбивается на чанки с перекрытием.

# Формат выходного результата
Программа выводит:

список найденных чанков с оценкой релевантности;

при использовании LLM — связный ответ со ссылками на источники.

# Проверка работоспособности
Минимальная проверка после запуска:

Убедиться, что LM Studio запущена и сервер активен.

Выполнить индексацию: go run cmd/docsearch/main.go --index.

Задать вопрос: go run cmd/docsearch/main.go --query "Что такое эмбеддинги?".

Убедиться, что вывод содержит релевантные фрагменты документов и оценки.

# Руководство пользователя
Индексация документов
Для начала работы необходимо поместить файлы .md или .txt в папку docs. После этого выполняется команда индексации, которая разбивает тексты на чанки, вычисляет эмбеддинги через LM Studio и сохраняет их в памяти.

Поиск по документации
Пользователь задаёт вопрос в командной строке с флагом --query. Система вычисляет эмбеддинг запроса, находит топ‑k самых похожих чанков по косинусной близости и выводит их с оценками релевантности. Если все оценки ниже заданного порога min_score, система сообщает, что ответа в документации не найдено.

# Безопасность
Конфигурационные параметры хранятся в файле configs/config.yml. Секретные ключи не используются в локальном режиме. При подключении внешних API ключи передаются через переменные окружения и не логируются. Локальный сервер LM Studio работает на локальном адресе и не доступен извне.
