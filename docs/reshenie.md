# Решение DocSearch

## Стратегия чанкования

Документы разбиваются на фрагменты с помощью умного чанкования:

1. Текст делится на секции по заголовкам Markdown (#, ##, ###)
2. Внутри секции текст режется по предложениям
3. Чанки создаются с перекрытием (параметр overlap_tokens из конфига)
4. Размер чанка ограничен max_tokens

Этот подход сохраняет смысловую целостность и не теряет контекст на границах.

## Выбор моделей

### Эмбеддинги
Используется модель `text-embedding-nomic-embed-text-v1.5` через LM Studio:
- Работает локально, не требует интернета
- Размер вектора 768
- Поддерживается через OpenAI-совместимый API

### LLM
Для генерации используется OpenRouter с моделью `openrouter/free`:
- Доступ к разным моделям через единый API
- Легко менять модель через конфигурацию

## Схема поиска

1. Вопрос пользователя превращается в вектор (эмбеддинг)
2. Вектор ищется в Qdrant по косинусной близости
3. Результаты фильтруются по порогу min_score
4. Берутся top_k самых похожих чанков
5. Чанки передаются в LLM для генерации ответа

## Формат промпта

Промпт для LLM содержит:
- Системную инструкцию (отвечать только по документам)
- Найденные чанки с указанием источника
- Историю диалога (последние 4-5 сообщений)
- Текущий вопрос пользователя

Формат ссылок: [источник: название_файла.pdf, страница N]

## Обработка случая "нет ответа"

Если после фильтрации по min_score не осталось ни одного чанка, возвращается ответ:
"В документации нет информации по этому вопросу"

## Известные ограничения

1. Требуется запущенный Qdrant и LM Studio
2. Поддерживаются только форматы: md, txt, pdf, html
3. Полный перебор при поиске (медленно при большом количестве чанков)
4. История чата ограничена 4-5 сообщениями
